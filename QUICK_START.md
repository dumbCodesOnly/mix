# Quick Start Guide\n\nGet the AI Platform Backend and Mobile Client running in minutes.\n\n## Prerequisites\n\n- **HuggingFace API Key**: Get one at https://huggingface.co/settings/tokens (free account required)\n- **For Backend**: Python 3.11+ and pip\n- **For Mobile**: Node.js 16+ and npm/yarn\n- **Optional**: Docker and Docker Compose for containerized deployment\n\n## Backend Setup (5 minutes)\n\n### 1. Clone and Navigate\n\n```bash\ncd /path/to/ai-platform\n```\n\n### 2. Create Virtual Environment\n\n```bash\npython -m venv venv\nsource venv/bin/activate  # On Windows: venv\\Scripts\\activate\n```\n\n### 3. Install Dependencies\n\n```bash\npip install -r requirements.txt\n```\n\n### 4. Configure Environment\n\n```bash\ncp .env.example .env\n```\n\nEdit `.env` and add your HuggingFace API key:\n\n```bash\nHF_API_KEY=your_huggingface_api_key_here\n```\n\n### 5. Run the Server\n\n```bash\npython -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000\n```\n\nThe API will be available at `http://localhost:8000`\n\n**API Documentation**: Visit `http://localhost:8000/docs` in your browser\n\n## Mobile Client Setup (5 minutes)\n\n### 1. Navigate to Mobile Directory\n\n```bash\ncd mobile_client\n```\n\n### 2. Install Dependencies\n\n```bash\nnpm install\n# or\nyarn install\n```\n\n### 3. Start Development Server\n\n```bash\nnpm start\n```\n\n### 4. Run on Device/Emulator\n\n**iOS (Mac only)**:\n```bash\nnpm run ios\n```\n\n**Android**:\n```bash\nnpm run android\n```\n\n**Web**:\n```bash\nnpm run web\n```\n\n**Or use Expo Go**:\n1. Install Expo Go on your mobile device\n2. Scan the QR code displayed in the terminal\n\n## Using Docker (Optional)\n\n### Start Everything with Docker Compose\n\n```bash\ncp .env.example .env\n# Edit .env and add your HF_API_KEY\n\ndocker-compose up\n```\n\nThe backend will be available at `http://localhost:8000`\n\n## Testing the API\n\n### Health Check\n\n```bash\ncurl http://localhost:8000/health\n```\n\nExpected response:\n```json\n{\n  \"status\": \"healthy\",\n  \"timestamp\": \"2025-12-22T10:30:00Z\"\n}\n```\n\n### Generate an Image\n\n```bash\ncurl -X POST http://localhost:8000/api/image \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"prompt\": \"A beautiful sunset over mountains\",\n    \"height\": 512,\n    \"width\": 512\n  }' \\\n  --output generated_image.png\n```\n\n### Text-to-Speech\n\n```bash\ncurl -X POST http://localhost:8000/api/tts \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"text\": \"Hello, this is a test of text-to-speech\"\n  }' \\\n  --output speech.wav\n```\n\n### Chat with LLM\n\n```bash\ncurl -X POST http://localhost:8000/api/llm \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"messages\": [\n      {\n        \"role\": \"user\",\n        \"content\": \"What is the capital of France?\"\n      }\n    ]\n  }'\n```\n\n### Generate Embeddings\n\n```bash\ncurl -X POST http://localhost:8000/api/embedding \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"text\": \"Sample text to embed\"\n  }'\n```\n\n## Configuring the Mobile App\n\n1. Open the app and navigate to the Home screen\n2. Tap \"Configure API\" button\n3. Enter your backend URL (e.g., `http://192.168.1.100:8000` for local network)\n4. The URL is saved for future sessions\n\n## Common Issues\n\n### Backend won't start\n\n**Issue**: `ModuleNotFoundError: No module named 'fastapi'`\n\n**Solution**: Ensure you've activated the virtual environment and installed dependencies:\n```bash\nsource venv/bin/activate\npip install -r requirements.txt\n```\n\n### HuggingFace API Key Error\n\n**Issue**: `401 Unauthorized` or `Invalid API key`\n\n**Solution**: \n1. Verify your API key at https://huggingface.co/settings/tokens\n2. Ensure it's correctly set in `.env`:\n   ```bash\n   HF_API_KEY=hf_xxxxxxxxxxxxxxxxxxxx\n   ```\n3. Restart the backend\n\n### Mobile app can't connect to backend\n\n**Issue**: Network error when trying to generate images\n\n**Solution**:\n1. Ensure backend is running: `curl http://localhost:8000/health`\n2. On mobile, use your computer's IP address instead of `localhost`\n3. Find your IP: `ipconfig` (Windows) or `ifconfig` (Mac/Linux)\n4. Configure app with: `http://YOUR_IP:8000`\n\n### Port already in use\n\n**Issue**: `Address already in use: ('0.0.0.0', 8000)`\n\n**Solution**: Use a different port:\n```bash\npython -m uvicorn app.main:app --port 8001\n```\n\n## Next Steps\n\n1. **Explore the API**: Visit `http://localhost:8000/docs` for interactive documentation\n2. **Read the README**: See `README.md` for detailed API documentation\n3. **Review Architecture**: Check `ARCHITECTURE.md` for system design\n4. **Deploy**: Follow `DEPLOYMENT.md` for production deployment\n5. **Mobile Development**: See `mobile_client/README.md` for mobile-specific setup\n\n## Performance Tips\n\n- **First request is slow**: Models are downloaded on first use (~2-5GB depending on model)\n- **Subsequent requests are faster**: Models are cached in memory\n- **Use smaller models**: For faster inference, specify smaller models\n- **Enable caching**: Mobile app automatically caches generated images\n\n## Resources\n\n- [FastAPI Documentation](https://fastapi.tiangolo.com/)\n- [HuggingFace Hub](https://huggingface.co/)\n- [React Native Documentation](https://reactnative.dev/)\n- [Expo Documentation](https://docs.expo.dev/)\n\n## Support\n\nFor issues or questions:\n\n1.
